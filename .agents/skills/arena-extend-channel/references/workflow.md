# Existing Channel Workflow

## Default sequence

1. Inspect the channel with `channel-get`.
2. Read the first page of contents with `channel-contents --per 24`.
3. Confirm the authenticated user can add content.
4. Infer the motif from titles, source domains, block types, and recurring subjects.
5. Use Premium search for complementary Are.na material and web search for fresh outside material.
6. Build a 6-item manifest that complements rather than clones the existing pattern.
7. Execute through `$arena-api-operator`.

## Heuristics

- If the channel already has many links but few native text notes, add a small text block.
- If the channel is heavy on theory, one playful or vernacular item can improve the notebook feel.
- If the channel already contains a very strong sub-theme, deepen it with one or two items and bend it with the rest.
- If the channel already contains the obvious canonical source, look for stranger or more sideways material instead of repeating the canon.
